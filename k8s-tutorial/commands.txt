Start a minikube cluster
    minikube start

See all api-resources / kinds in the cluster
    kubectl api-resources

See all nodes in the cluster
    kubectl get nodes

See all namespaces 
    kubectl get namespaces

See Pods in a namespace
    kubectl get pods -n <namespace>

See Pods in all namespaces
    kubectl get pods -A 

See which cluster kubectl is connected to:
    kubectl config get-contexts

See verbose output
    kubectl get ns -v6
    kubectl get ns -v10

Create a resource
    kubectl create -f nginx1.yaml

Use labels to search for Pods
    kubectl get pods -lapp=nginx
    kubectl get pods -lapp=nginx,type=lb
    kubectl get pods -lapp=nginx,type=lb,type1=abc

Update a resource
    kubectl apply -f nginx1.yaml

Delete a resource
    kubectl delete -f nginx1.yaml

Describe a resource
    kubectl describe pod <podname> -n <namespace>

Get JSON output
    kubectl get pod <podname> -n <namespace> -o json

Expose the Pod to the outside world
    kubectl create -f nginx-service.yaml

Get the Pods's URL
    minikube service nginx-service --url

Understand Spec for a kind
    kubectl explain <kind>.spec
    e.g.: kubectl explain Pod.spec.container.....

Exec into a Pod
    kubectl exec -it <Pod name> -- /bin/bash

Run Prometheus on your machine as a container and scrape Kubernetes metrics from your minikube cluster 
    docker run -d --name prometheus \
        --network minikube \
        -p 9090:9090 \
        -v ./monitoring/prometheus-k8s.yml:/etc/prometheus/prometheus.yml \
        -v ~/.minikube:/minikube-certs:ro \
        prom/prometheus

Deploy an instrumented application and pull metrics in Prometheus
    Terminal 1: 
    - Connect your Docker CLI to Minikube Docker daemon
    - Instrument your application with Prometheus Counter (name: custom_request_count)
    - Deploy your instrumented application (use NodePort type of Service with Port 32767)
    - Port forward localhost port 8081 to port 80 of your app's Service
        - kubectl port-forward service/<app-service> 8081:80 
   
    Terminal 2:
    - Run Prometheus as a Docker container on the minikube network. Make sure to use <minikube ip>:32767 as the target in prometheus.yml
    docker run -d --name prometheus --network minikube -p 9090:9090 -v ./<your-prometheus.yml>:/etc/prometheus/prometheus.yml prom/prometheus

    Access application endpoints 
    - use curl from terminal 3 or 
    - use browser

    Access Prometheus:
    - In browswer, open http://localhost:9090
    - Go to Status -> Target health and verify that your configured Target is up (green)
    - Go to Query -> check custom_request_count_total metric
      - Type "custom_request_count_total" in the Query box and hit Execute
      - Check the Graph view
    
    Experiment:
    - As you hit the application endpoints, Prometheus Graph view should show increasing graph

Multi-team Multi-tenancy Experiment
-------
Cluster admin steps
--------------------
wget https://raw.githubusercontent.com/cloud-ark/kubeplus/master/requirements.txt
wget https://raw.githubusercontent.com/cloud-ark/kubeplus/master/provider-kubeconfig.py
python3 -m venv venv
source venv/bin/activate
pip3 install -r requirements.txt
apiserver=`kubectl config view --minify -o jsonpath='{.clusters[0].cluster.server}'`
echo $apiserver
kubectl create ns ns1; kubectl create ns ns2
python3 provider-kubeconfig.py -s $apiserver -x minikube -c tenant1 -f tenant1.json create ns1
python3 provider-kubeconfig.py -s $apiserver -x minikube -c tenant2 -f tenant2.json create ns2

Experiment
-----------
Steps performed by the teams
-----------------------------
KUBECONFIG=tenant1.json kubectl create deployment nginx --image=nginx:latest -n ns1
KUBECONFIG=tenant1.json kubectl run pod nginx --image=nginx:latest -n ns1
KUBECONFIG=tenant2.json kubectl create deployment nginx --image=nginx:latest -n ns2
KUBECONFIG=tenant2.json kubectl create deployment nginx --image=nginx:latest -n ns1
KUBECONFIG=tenant1.json kubectl get pods -n ns2 (may be okay)
KUBECONFIG=tenant1.json kubectl delete pod <podname> -n ns2 (should be disallowed)

Check permissions:
-------------------
1) What permissions does tenant1 service account have at cluster scope?
kubectl auth can-i --list --as=system:serviceaccount:ns1:tenant1

2) What permissions does tenant1 service account have in namespace tenant2?
kubectl auth can-i --list --as=system:serviceaccount:ns1:tenant1 -n tenant2
